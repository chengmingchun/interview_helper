手撕算法五步

面试时别一上来写代码，先用这五步稳住节奏。

1. 复述题意确认输入、输出、是否连续、是否原地、边界是什么。
2. 找状态/指针数组题找状态，链表题找指针，树题找递归语义。
3. 跑样例拿最小例子手推一轮，验证状态怎么变。
4. 写模板先写骨架，再填转移/指针变化，避免边写边想。
5. 补复杂度主动说时间、空间、边界和变体，给面试官安全感。

八股补充速记

这些不是算法题，但仍然是 Java 后端面试高频追问。

线程安全单例

DCL 必须 `volatile` 防止指令重排；静态内部类靠类加载保证线程安全；枚举单例最稳，能防反射和反序列化破坏。

JVM 内存区域 + GC Roots

线程私有：程序计数器、虚拟机栈、本地方法栈。线程共享：堆、方法区。GC Roots 常见有栈中引用、静态属性、常量、JNI 引用、活跃线程等。

索引失效 + Explain

失效常见：函数、隐式转换、左模糊、最左前缀不满足、低区分度。Explain 重点看 type、key、rows、Extra。

Redis 缓存一致性

常用 Cache Aside：读缓存，未命中查 DB 回填；写先更新 DB，再删除缓存。删除失败要重试，并发旧值回填可用延迟双删/TTL/版本兜底。
