Redis 专题 / Sentinel · Cluster · Production

Redis 高可用不是“多放几台”，而是主从、选主、路由、迁移和一致性的取舍。

面试里问 Redis 集群，真正想听的是：哨兵解决高可用，Cluster 解决分片扩容；主从复制是基础，但异步复制会带来数据丢失窗口；客户端、故障转移、热点 key、脑裂和缓存一致性都要讲清。

Animated Redis Flow

从一次写入到故障转移怎么流动

1. 写 Master客户端写主库，Redis 先返回业务成功。
2. 异步复制写命令传播到 replica，存在复制延迟。
3. 哨兵判活主观下线到客观下线，quorum 降低误判。
4. 选主切换挑复制偏移量更新的从库晋升。
5. Cluster 路由key 算 slot，MOVED/ASK 完成重定向。
6. 一致性补偿承认窗口，用重试、TTL、binlog 兜底。

当前重点：客户端写 master 成功不代表 replica 已经同步，Redis 高可用默认不是强一致。
