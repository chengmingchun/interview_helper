Spring 面试的主线：容器创建对象，AOP 增强对象，事务保护边界，Boot 自动装配工程。

不要把 Spring 背成散点。把它看成一条请求链路：启动时扫描配置生成 BeanDefinition，容器创建 Bean 并处理依赖，AOP 用代理织入增强，请求进入 DispatcherServlet，业务方法通过事务代理完成提交或回滚。

2. IoC、Bean 生命周期与循环依赖

IoC 面试最常追问：Bean 怎么来的？生命周期有哪些扩展点？循环依赖为什么能解决一部分？

1. 扫描配置XML、注解、配置类被解析。
2. BeanDefinition记录类、作用域、依赖。
3. 实例化构造方法创建原始对象。
4. 属性填充@Autowired 注入依赖。
5. 初始化Aware、PostProcessor、init。
6. 代理/使用/销毁AOP 代理进入容器。

Bean 生命周期关键扩展点

  1. 实例化：构造方法执行。
  2. 依赖注入：填充属性、处理 @Autowired。
  3. Aware：注入 BeanName、BeanFactory、ApplicationContext。
  4. BeanPostProcessor before。
  5. InitializingBean / init-method。
  6. BeanPostProcessor after，AOP 代理常在这里生成。
  7. DisposableBean / destroy-method。

三级缓存解决什么循环依赖

缓存含义
singletonObjects一级缓存，完整单例 Bean。
earlySingletonObjects二级缓存，提前暴露的早期 Bean。
singletonFactories三级缓存，ObjectFactory，可提前创建代理。

只能解决默认单例、属性注入的循环依赖；构造器循环依赖无法解决，因为对象还没创建出来。

3. AOP 与代理机制

AOP 解决什么

把事务、日志、权限、缓存、监控这类横切逻辑从业务代码里抽离。核心概念：切面、切点、通知、连接点、织入。

Controller -> Proxy(Service)
  -> TransactionInterceptor
  -> target.business()
  -> commit / rollback

JDK 代理 vs CGLIB

方式特点限制
JDK 动态代理基于接口生成代理类目标类需要接口
CGLIB生成目标类子类final 类/方法不能代理

Spring Boot 默认倾向 CGLIB 代理类，但是否使用接口代理也与配置和场景有关。面试要强调：AOP 生效的本质是“调用代理对象”。

自调用失效：同一个类内部 this.method() 调用不会经过代理对象，所以事务、缓存、异步等 AOP 增强不会生效。解决：拆到另一个 Bean、通过代理对象调用、或使用 AspectJ。

4. 事务：传播行为、隔离级别与失效场景

Spring 事务不是魔法，本质是 AOP 代理 + TransactionInterceptor + 数据库事务。

常见传播行为

  • REQUIRED：有事务就加入，没有就新建，最常用。
  • REQUIRES_NEW：挂起外部事务，开启新事务。
  • NESTED：嵌套事务，依赖保存点。
  • SUPPORTS：有事务就加入，没有就非事务执行。
  • MANDATORY：必须存在事务，否则异常。

事务失效高频点

  • 方法不是 public。
  • 同类内部自调用。
  • 异常被 catch 后没有抛出。
  • 默认只回滚 RuntimeException/Error，受检异常需配置 rollbackFor。
  • 数据库引擎不支持事务，或方法没有被 Spring 管理。
@Transactional(rollbackFor = Exception.class)
public void pay() {
  updateOrder();
  sendEventByOutbox();
}

5. Spring MVC 请求链路

把 MVC 背成一条链：请求进 DispatcherServlet，找 Handler，执行拦截器和 Controller，返回 ModelAndView 或 JSON。

阶段组件面试说法
入口DispatcherServlet前端控制器，统一接收请求。
找处理器HandlerMapping根据 URL、HTTP 方法、注解找到 Handler。
适配执行HandlerAdapter适配不同 Handler 方法签名并执行。
参数/返回ArgumentResolver / ReturnValueHandler解析 @RequestBody、@PathVariable，处理 JSON 返回。
拦截HandlerInterceptor / FilterFilter 更靠近 Servlet，Interceptor 更靠近 Spring MVC。

6. Spring Boot 自动配置

自动配置怎么生效

  1. @SpringBootApplication 组合了 @SpringBootConfiguration、@EnableAutoConfiguration、@ComponentScan。
  2. @EnableAutoConfiguration 导入 AutoConfigurationImportSelector。
  3. 读取 AutoConfiguration.imports 或旧版 spring.factories。
  4. 按 @ConditionalOnClass、@ConditionalOnMissingBean、@ConditionalOnProperty 条件装配。

Starter 是什么

Starter 是依赖聚合 + 自动配置入口。它把某类功能需要的依赖、默认配置和条件装配打包，让业务项目引入即可使用。

starterauto-configurationconditional
排障技巧：自动配置没生效时，看依赖是否存在、条件注解是否满足、是否已有用户自定义 Bean、配置属性是否正确，可打开 condition evaluation report。
